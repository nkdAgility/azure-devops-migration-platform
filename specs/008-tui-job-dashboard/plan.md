## Summary

Build the interactive Terminal UI that lets operators monitor migration jobs in real time. The TUI connects to an already-running control plane (local Aspire stack or remote) and displays a **single-screen three-panel layout**:

1. **Job List** (left ~30%) — scrollable table of all visible jobs with live state, auto-refreshing every 10 s
2. **Metrics Panel** (center ~35%) — polled `MetricSnapshot` from `GET /jobs/{jobId}/telemetry` for the selected job; empty placeholder when no job is selected
3. **Log Panel** (right ~35%) — toggles between Progress event SSE stream and Diagnostics SSE stream for the selected job; Tab key switches mode; mode indicator in panel header

Selecting a job in the Job List updates the Metrics and Log panels in place — no navigation or separate screen. Additionally, every CLI migration command that submits a job (`export`, `import`, `migrate`) is updated to print the Job ID and control plane URL immediately after submission. A `--job <id>` flag pre-selects a job on launch.

**Technology decision**: Terminal.Gui v2 Beta (`2.0.0-beta.*`, .NET 10.0 native) for all TUI views. Existing Spectre.Console views (`DiagnosticsPanel`, `TelemetryPanel`, `TelemetryPoller`) are untouched — they serve the CLI follow-mode pipeline.

### Terminal.Gui v2 Application pattern in `TuiCommand`

```csharp
protected override async Task<int> ExecuteInternalAsync(
    CommandContext context, TuiCommandSettings settings, CancellationToken ct)
{
    var url = ResolveUrl(settings.Url);
    // Attempt connection — TUI NEVER starts LocalStackHost
    if (!await TryReachControlPlaneAsync(url, ct))
    {
        AnsiConsole.MarkupLine($"[red]✗[/] Cannot reach control plane at [bold]{url}[/].");
        AnsiConsole.MarkupLine("[grey]  Run a migration command first (which starts the local stack), or supply --url.[/]");
        return 1;
    }

    await using var host = await CreateTuiHost(settings, url);
    var client = host.Services.GetRequiredService<ControlPlaneClient>();

    using IApplication app = Application.Create().Init();

    var mainView = new TuiMainView(client, app);
    if (settings.Job is { } jobIdStr && Guid.TryParse(jobIdStr, out var jobId))
    {
        mainView.PreSelectJob(jobId);  // pre-selects row; populates Metrics + Log immediately
    }
    app.Run(mainView);

    return 0;
}
```

### Job submission output (shared helper)

// On ControlPlaneCommandBase or as a static method in a shared helper:

## Open Items (to be resolved during implementation)

1. **`JobRecord.State` mutation** — the current `MigrationJob` is an immutable record. `JobRecord.State` must be mutable string (or use a separate `ConcurrentDictionary<Guid, string>` in `JobStore`). Prefer the dictionary to keep `JobRecord` immutable.
2. **Auth forwarding in `TuiCommand`** — ✅ Resolved: `TuiCommand` calls `CreateHost(settings)` (the same `ControlPlaneCommandBase` method used by all other commands). This ensures the `HttpClient` registered in the DI host has the correct Entra/Windows auth handler. No separate `CreateTuiHost` needed.
3. **`Application.Invoke` from async SSE loop** — verify in prototype that `Application.Invoke` is safe to call before `app.Run` returns; some v2 betas have constraints on post-dispose invocation.
